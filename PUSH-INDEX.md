# Kineo — índice de PUSHes

## Regra de nomenclatura

- Cada release deve usar o nome `PUSH-XX-RELEASE.md` na raiz do projeto.
- O número nunca deve ser reutilizado, mesmo quando um PUSH for cancelado.
- O arquivo deve registrar objetivo, escopo, validação, commit, deploy e resultado em produção.
- Ao iniciar um novo release, usar o próximo número livre deste índice.
